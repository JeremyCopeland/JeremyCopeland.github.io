Overview

Welcome to our grader feature analysis tool. This tool is designed to solve a very precise problem: We set out to find all of the obvious mathematical statements in each student's submission and then check whether these statements are correct or incorrect in the context of the problem. You can activate this tool by clicking the Analyze Submission button, if the tool is available for the given submission. How does it work, you ask? Well, we begin by giving the machine a rubric that explains to it what things are true in the problem we are grading. In many problems, like when there is a single unique answer to the the problem, the machine tests that every statement the student makes is consistent with the answer. For example, if the question is: Find all positive solutions to x+y=2 x-y=0, then there is only one solution, namely x=y=1. When a student says something correct, like x=1, we mark it as such. The correct statement needn't be <emph>interesting</emph> to be correct. Take x^2+2xy=3 as an example, or 3+6=9, which are both true, but it is still up to the grader to know whether either statement is relevant. If the student says something false, such as x+y=3, then we mark that as well. Furthermore, the machine understands that an expression cannot be evaluated, but you may still want to know something about it. So if you hover your mouse over x/y or 3x, you will learn its value and you will also be shown other expressions that have the same value. The machine is also capable of dealing with chains of statements as in 2+2=2*2=2^2=2/2. It can parse a limited number of non-equation-looking text as well. For example, 2x is equal to 7 minus 5y. Finally, we also have the ability to check the negations of statements. For example, x is not equal to 2x. We mark this in pink because it is a true statement, but not a very powerful one. It turns out that it is much easier to make a false statement than to make a true statement. For example, there's only one value that x can be, namely x=1. However, there are infinitely many inequality statements that can be made, x≠2, x≠3, x≠4, etc.

Configurations

However, when working with many problems, and with geometry in particular, there are usually infinitely many ways in which a statement can be true. For example, in the diagram, we have given the server a rubric that tells it that ABC is an arbitrary right triangle. Instead of working with formal mathematical statements, the machine works by sampling the full configuration space. If you hover over the phrase triangle ABC is similar to triangle ACH, you will see that the machine believes this is a true statement because it is true for its three chosen sample diagrams. Now you can be confident about these statements. Be aware that even if the student gets the wrong ordering, such as triangles ACH and BCH are similar, the machine will recognize the error and also point out the incorrect ordering of the vertices. We are also able to handle negations, as in angles ABC and BAC are not congruent, or triangles ACH and BCH are not congruent.

Declarations

We are also able to handle new variable declarations to some extent. Here we define some new points. We let G be the midpoint of AC. and we also let the midpoint of PG be X. Finally, let AE and XB intersect at O. Now that we have these points, they are usable, as in angle OAX = angle ODX or angle OAX is not congruent to angle ODX.

Best Practices

So, should the analysis server eventually replace the human work we do to verify student claims? Absolutely not, though it should greatly help with that process. This bot has many strengths, and it should save a lot of time with certain student submissions. But the tool has many limitations: combinatorics is hard for it to parse, students may write in insufficiently clear language, and proof by contradiction is a whole can of worms. Ensuring that a student's solution even includes the necessary steps will remain entirely a human domain (for now). The tool will be great at considering geometric statements, assigning new variables, and when it thinks it knows enough to make an assessment about a statement, it is very accurate. Play around with the bot, and get a sense of how you can best employ it without over-relying on it. First, read the student submission. If the student wrote enough expressions that the AI can parse, you can skim, trusting that you’ll verify them with the bot’s help. For any statement marked as true, it’s sufficient to give a sanity check on the math and make sure it looks like it's written in a way the bot can handle. If you're not sure, always tend on the side of double checking. We hope this makes grading easier, faster, and/or more human. If you get the chance to play with it, let us know any successes or bugs on the School Staff Forum. Happy problem solving!
